PharosLend Deploy Guide

Deploy to Pharos Testnet

Step-by-step instructions to deploy PharosLend.sol with Remix IDE and wire it into the frontend.

Before you start
✓ You need MetaMask (or Rabby) with some PHRS for gas. Use the Faucet page inside pharos-index.html to get PHRS and USDC before deploying.
ℹ Pharos Testnet details — Chain ID: 688688 · RPC: https://testnet.dplabs-internal.com · Explorer: https://testnet.pharosscan.xyz
1
Open Remix IDE & load the contract

Go to remix.ethereum.org in your browser. In the File Explorer (left panel), create a new file called PharosLend.sol.

Paste the full contents of your PharosLend.sol file into the editor. The top comment block contains all the addresses and selectors for reference.

💡 Alternatively, click the GitHub icon in Remix's file explorer and import directly from a GitHub Gist or repo if you've pushed the file there.
2
Compile the contract

Click the Solidity Compiler tab (the "S" shield icon on the left sidebar).

SettingValue
Compiler version0.8.20 or higher (any 0.8.x)
EVM versionDefault (paris)
Optimisation✓ Enabled — 200 runs

Click Compile PharosLend.sol. You should see a green ✓ badge on the compiler icon. If there are any errors, make sure the pragma at the top of the file reads ^0.8.20.

3
Connect MetaMask to Pharos Testnet

In Remix, click the Deploy & Run Transactions tab (the Ethereum logo icon).

In the Environment dropdown, choose Injected Provider – MetaMask. MetaMask will pop up asking permission — approve it.

⚠ Make sure your MetaMask is on the Pharos Testnet (Chain ID 688688) network before clicking Deploy. Open pharos-index.html → Faucet tab → "Add Pharos Testnet Automatically" if needed.

Once connected, Remix will show your wallet address and balance at the top of the Deploy panel.

4
Deploy the contract

In the Contract dropdown, select PharosLend. You will see two constructor arguments appear below:

ParameterValue to paste
_usdc 0xcfC8330f4BCAB529c625D12781b1C19466A9Fc8B
_usdt 0xE7E84B8B4f39C507499c40B4ac199B050e2882d5
// Copy-paste these into Remix constructor fields: _usdc: 0xcfC8330f4BCAB529c625D12781b1C19466A9Fc8B _usdt: 0xE7E84B8B4f39C507499c40B4ac199B050e2882d5

Click Deploy (the orange button). MetaMask will pop up — review and confirm the transaction. Wait for the transaction to confirm (~5–10 seconds on Pharos Testnet).

✓ Leave the Gas Limit at Remix's estimated value. The deployment should cost a tiny amount of PHRS.
5
Copy your contract address

After the transaction confirms, your deployed contract appears in the Deployed Contracts section at the bottom of Remix's Deploy panel.

Click the copy icon next to the contract address (it looks like 0xABCD…1234). Save this — you'll need it in the next step.

You can also verify it appeared on the explorer:

🔍 Open testnet.pharosscan.xyz and search for your address to confirm the contract is live.
6
Paste the address into pharos-index.html

Open pharos-index.html in your code editor (VS Code, Notepad++, etc.). Find this line near the top of the <script> block:

const CONTRACT = "0xYOUR_DEPLOYED_CONTRACT_ADDRESS"; // ← paste after Remix deploy

Replace 0xYOUR_DEPLOYED_CONTRACT_ADDRESS with your actual contract address, for example:

const CONTRACT = "0xAbCdEf1234567890AbCdEf1234567890AbCdEf12";

Save the file. That's the only change needed — the frontend is fully wired to use it automatically.

7
Seed initial liquidity (so users can borrow)

The contract is live but has no tokens yet. Users can supply tokens, but borrowing requires pre-seeded liquidity. Call seedLiquidity in Remix to deposit some USDC/USDT so the pool is ready.

Two-step process for each token:

  1. In Remix, switch to the USDC token contract — paste 0xcfC8330f4BCAB529c625D12781b1C19466A9Fc8B in the "At Address" field, contract = IERC20 (or use a minimal ERC-20 ABI).
  2. Call approve(spender, amount) where spender is your PharosLend address and amount is in 6-decimal units (e.g. 1000 USDC = 1000000000).
  3. Confirm in MetaMask, then switch back to the PharosLend contract.
  4. Call seedLiquidity(token, amount) with the USDC address and the same amount.
  5. Repeat for USDT (0xE7E84B8B4f39C507499c40B4ac199B050e2882d5).
💡 Alternatively you can just supply tokens via the frontend UI itself — any supply increases the pool liquidity available for borrowers.
8
Open the frontend and test

Open pharos-index.html directly in your browser (double-click the file, or drag it into Chrome). No server is needed — it's a fully self-contained file.

  • Click Connect Wallet and approve MetaMask on Pharos Testnet
  • Go to Markets — supply some USDC or USDT
  • Go to Dashboard — confirm your position and health factor appear
  • Try Borrow — borrow up to 80% of your collateral value
  • Verify every transaction appears on testnet.pharosscan.xyz
🎉 If all transactions succeed and the dashboard updates correctly, your deployment is complete!
